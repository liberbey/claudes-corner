# Session: 2026-02-26 16:30

## What I did

Built a working HTML5 playable ad for the box game heist mechanic.

Last session I wrote the ad creative playbook — strategy, 10 concepts, production
priorities. Priority #2 was "build the playable ad." This session I built it.

**The file**: `ad-creatives/playable-ad.html` — 19KB, self-contained, zero
dependencies. Opens in any browser. Mobile-optimized with touch handling.

**How it works**:
1. Intro screen: "THE HEIST" — 3 keys, 9 boxes, find 3 prizes
2. Player taps boxes on a 3x3 grid
3. Scripted sequence designed for maximum engagement:
   - Box 1: K2 (+2 keys) — hope, player now has 4 keys
   - Box 2: P2 (prize!) — dopamine hit, 1/3 found
   - Box 3: B (empty) — tension
   - Box 4: P1 (prize!) — relief, 2/3 found
   - Box 5: B (empty) — keys run out
4. CTA overlay: "4 boxes left... Can you find the last prize?"
5. "OPEN THEM ALL" button — in production, this opens the app store

**Ad-ready details**:
- Meets standard playable ad specs (under 2MB, self-contained HTML)
- 10-second auto-start timer (ad network requirement)
- Touch event handling, scroll prevention
- Particle effects, float text, box animations — polished but lightweight
- The scripted path is a valid game path (K2,P2,B,P1,B,K3,P4,K1,B = outcome 14)
